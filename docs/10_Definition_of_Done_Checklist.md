# Definition of Done (DoD) Checklist

The project is considered complete and ready for handover when the following criteria are met:

## Documentation
- [ ] Requirements Analysis and Workflow Mapping Report delivered.
- [ ] Technical Architecture Document delivered.
- [ ] Data Protection and Security Plan delivered.
- [ ] Testing and QA Plan delivered.

## DHIS2 Configuration
- [ ] Complete DHIS2 metadata dictionary compiled.
- [ ] Importable DHIS2 metadata JSON package generated and tested.
- [ ] TB Tracker program structure fully configured (Registration + 8 Program Stages).
- [ ] Option sets and Data elements configured according to national HMIS guidelines.
- [ ] Program rules and Validation rules implemented and tested.
- [ ] Program indicators and DHIS2 Dashboards configured.
- [ ] User roles, User Groups, and Sharing Settings configured.

## Custom Web App (if deployed)
- [ ] React-based dashboard UI deployed and integrated with DHIS2 Web API.
- [ ] Responsive design verified on desktop and tablet.
- [ ] Role-based navigation functioning correctly.

## Testing & Quality Assurance
- [ ] Unit testing on metadata import successful.
- [ ] Program rules fire correctly for alerts and required fields.
- [ ] Role restrictions (OrgUnit isolation) verified.
- [ ] Mobile/offline compatibility verified with DHIS2 Android Capture.
- [ ] UAT feedback documented, triaged, and resolved.

## Deployment & Training
- [ ] Pilot deployment plan executed successfully.
- [ ] Training materials (presentations, user guides, job aids) delivered.
- [ ] System Administrator Guide and Deployment Guide delivered.
- [ ] Mock data generated and evaluation demonstration script prepared.

## Support & Handover
- [ ] Maintenance and Support SOP (12-Month Support Plan) defined.
- [ ] Final handover package signed by Plan International and Ministry of Health.
